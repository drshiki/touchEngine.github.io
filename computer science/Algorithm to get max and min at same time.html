Algorithm to get max and min at same time

Yesterday I saw a problem which requires the maximum and minimum of an array, we have a solution with 2(n-1) times comparison, but we need a less comparison times' algorithm. I think most people come up with the solution held the complexity 2n below in the first time they meet the problem:

	max = min = a[1];
	for i from 2 to n:
		if (max < a[i]):
			max = a[i];
		if (min > a[i]):
			min = a[i];

This algorithm conducts the 2(n-1) times of comparison. We use the C to represent the amount of the comparisons, then we get that Cbest = Cavg = Cworst = 2(n-1), it was a terrible solution in practical engineering, however, from my point of view, this kind of brute force algorithms would be good start points. Let's try to optimize the algorithm, because the similar process between getting the maximum and getting the minimum, I would think whether I can use some information come from previous process, after learning the KMP algorithm I knew that I could omit some unnecessary steps when we have gotten some information in the first process, this thinking is important, many other algorithms also contain this thinking generally. Back to this problem, look at the code it is easy to find out the fact that if max < a[i] then the condition a[i] >= min must true, so we needn't compare a[i] and min when max < a[i], now we get the initial optimizing:

	max = min = a[1];
	for i from 2 to n:
	    if (max < a[i]):
	        max = a[i];
	    else：
	        if (min > a[i]):
	            min = a[i];

The above holds the Cavg = Cbest = (n-1)+(n-1)/2=(3(n-1))/2, and the worst circumstance 2(n-1), later I found a better algorithm which keeps the comparison times Cavg = (3(n-2))/2+1 when the n-1 is odd and Cavg=(3(n-1))/2 when the n-1 is even, the algorithm at first comparison doesn‘t compare the initial max(min) with the next number, it firstly does compare between the next two numbers, then does compare between the max and the bigger one of the two numbers, does compare between the min and the smaller one, the detail is written in following:

	max = min = a[1];
	int start = 2;
	if((n-1)%2 == 1)://different handlings of the odd and even 
	    if(a[2] > max)://this is one of time of comparison
	        max = a[2];
	    else:
	        min = a[2];
	    start = 3;
	for i from stat to n by step 2:
	    if(a[i]>a[i+1]):
	        temp_max = a[i]；
	        temp_min = a[i+1];
	    else
	        temp_min = a[i+1];
	        temp_min = a[i];
	    if(max < temp_max):
	        max = temp_max;
	    if(min < temp_min):
	        min = temp_min;

Actually, you can get the biggest and the smallest of next three numbers to compare the max and the min, and it also works better than the first solution but worse than when we choose the two numbers.
