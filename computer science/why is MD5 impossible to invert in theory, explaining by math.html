why is MD5 impossible to invert in theory, explaining by math

We usually hear that MD5 could not be inverted to get the input value by the hash value, because it is well know thing, however have you known why that? whether it can be proved formally? MD5 is a kind of hash function to get fixed length hash value, and they are easily to be proved they can not be inverted in theory, owe it all to the prove method of existence of an inverse of an original function.

We assume that MD5 can be inverted, then the each of the hash values reflects an unique input string, obviously it results false. folling is the function picture of MD5 hash function, because it is impossible that MD5 function is a monotone function in infinite positive interval. from 6216 to +∞ each of all strings will be computed to get a hash value which in the set of hash value of string 0 to string 6216.

So that we could come a conclusion that all hash functions which to get a fixed length hash value can be inverted and all they can discover collision in some conditions.
