Loop Invariant

I met the loop invariant when I read the chapter 2 of CLRS, and in order to make it clear what it is I consulted some related material.

what the loop invariant is, according to my understanding, the most point is loop invariant is a property of a loop essentially, it is generally used to comfirm the loop's effect.

I know that some people of my group who work on formal program verification use it, but I don't know more of their work.

loop invariant is the property what the loop holds before its first iteration and also the exit, and keep in every iteration, it may be ambiguous, let's see a example:

				int i = 0;
				// (i <= 4) == true
				while(i<4){
				// (i <= 4) == true
					i++; 
				//(i <= 4) == true
				}
				//(i <= 4) == true

in above loop, throughout the loop it holds that i <= 4, it is one of the loop invariant of the loop. I believe you have understood that there aren't only one loop invariant in this loop, yea, you are right, such as the predicate i < 5, i < 6, i >= 0 etc, also are loop invariant of the loop, but we often use which is useful for our problem. sometimes the loop invariant we need is not relateed to the condition expression or very easy to be described by an expression, the condition expression if exisit always is included into the loop invariant.

lets's see the description in CLRS:

Initialization: It is true prior to the first iteration of the loop.

Maintenance: If it is true before an iteration of the loop, it remains true before the next iteration.

Termination: When the loop terminates, the invariant gives us a useful property that helps show that the algorithm is correct.
