Asymptotic notation

I intend to explain three kinds of asymptotic notation including \(\Theta,O,\Omega\). Following will diplay both informalized definition and formalized definition. There also will talk about \(\omega\)-notation and \(\omicron\)-notation without detail.

\(\Theta\) to describe asymptotic tight bound

\(O\) to describe asymptotic upper bound

\(\Omega\) to describe asymptotic lower bound

\(o\) to describe upper bound that is not asymptotically tight

\(\omega\) to describe lower bound that is not asymptotically tight

These asymptotic notations aren't privileges of algorithm complexity but apply to functions.

For example, the worst-case run time of insertion sort was \(T(n) = ax^2 + bx + c\), then this function can be writting as \(\Theta(n^2)\), hence \(\Theta(n^2)\) can characterize the worst-case run time of insertion sort.

How to count the run time of an algorithm

Let's see some specific case :

This an insertion sort algorithm in c like programming language.

n is input size of this case

\(c_i\) is the executing cost of line i

\(t_i\) is executing times of in each different value i

T(n) is the total time of this insertion sort

0                                           cost    times
1   int i = 1;                              c1      1
2   for (; i < n;) {                        c2      n
3       key = a[i];                         c3      n-1
4       j = i - 1;                          c4      n-1
5       for (;j >= 0 && key < a[j];){       c5      t1+t2+t3+...+t(n-1)
6           a[j + 1] = a[j];                c6      (t1-1)+(t2-1)+...+(t(n-1)-1)
7            j--;                           c7      (t1-1)+(t2-1)+...+(t(n-1)-1)
8       }                                   c8      n-1
9       a[j + 1] = key;                     c9      n-1
10       i++;                               c10     n-1
11  }                                       c11     n-1

\[T(n)=c_1+c_2n+c_3(n-1)+c_4(n-1)+c_5\sum_{i=1}^{n-1}t_i+c_6\sum_{i=1}^{n-1}(t_i-1)+c_7\sum_{i=1}^{n-1}(t_i-1)+c_8(n-1)+c_9(n-1)+c_{10}(n-1)+c_{11}(n-1)\]

Because we assume that line 8 and line 11 aren't executable, thus

\[T(n)=c_1+c_2n+(c_3+c_4+c_9+c_{10})(n-1)+c_5\sum_{i=1}^{n-1}t_i+(c_6+c_7)\sum_{i=1}^{n-1}(t_i-1)\]

Further simplify

\[T(n)=(c_2+c_3+c_4+c_9+c_{10})n-(c_3+c_4+c_9+c_{10}-c_1)+c_5\sum_{i=1}^{n-1}t_i+(c_6+c_7)\sum_{i=1}^{n-1}(t_i-1)\]

In this insertion sort, the best-case is to input a sorted array, then the line 5 just will execute for n-1 times, and the line 6, line 7,line 8 won't execute any more. We get the best-cast run time is :

\[T(n)=(c_2+c_3+c_4+c_5+c_9+c_{10})n-(c_3+c_4+c_5+c_9+c_{10}-c_1)\]

And obvious \(T(n)\) is a linear function, so we can use \(ax+b\) expressing \(T(n)\), because we only care the order of growth, when value of n is large, those lower-order terms and the constant coefficient of leading term are not important to our problem, so we ignore lower-order terms and the constant coefficient of leading term in the \(T(n)\) to left with \(n^2\), hence we say the insertion sort has a best-case run time of \(\Omega(n)\).

The worst-case is to input a reverse sorted array, then :

\[\sum_{i=1}^{n-1}t_i = \frac{(3+n)(n-1)}{2} = \frac{n^2+2n-3}{2}\]

\[\sum_{i=1}^{n-1}(t_i-1) = \frac{(n+1)(n-1)}{2} = \frac{n^2-1}{2}\]

\[c_5\sum_{i=1}^{n-1}t_i+(c_6+c_7)\sum_{i=1}^{n-1}(t_i-1)=(\frac{c_5}{2}+\frac{c_6}{2}+\frac{c_7}{2})n^2+c_5n-\frac{3c_5+c_6+c_7}{2}\]

\[T(n)=(\frac{c_5}{2}+\frac{c_6}{2}+\frac{c_7}{2})n^2+(c_2+c_3+c_4+c_5+c_9+c_{10})n-(c_3+c_4+\frac{5c_5}{2}+\frac{c_6}{2}+\frac{c_7}{2}+c_9+c_{10}-c_1)\]

Then we follow the above ignoring tenet, to left with \(n^2\), we could say insertion sort has the worst-case run time \(O(n^2)\), maybe you have found that I use the \(O\) instead of above \(\Omega\). yep, because they have different meaning.

We can make a conclusion from above content, we use the \(O\) to describe the worst-case and use \(\Omega\) to describe the best-case in algorithm complexity. however we use \(\Theta\) less, its meaning will be explain in following, this table and the graph should be useful to comprehend those notations for you.

Formalized definition

\(\Theta(g(n))= \{f(n) : \exists c_1,c_2,n_1 \in Q^+ \bigwedge n \geq n_1\rightarrow 0 \leq c_1g(n) \leq f(n) \leq c_2g(n)\} \)

\(O(g(n))= \{f(n) : \exists c_1,n_1 \in Q^+ \bigwedge n \geq n_1\rightarrow 0 \leq f(n) \leq c_1g(n)\} \)

\(\Omega(g(n))= \{f(n) : \exists c_1,n_1 \in Q^+ \bigwedge n \geq n_1\rightarrow 0 \leq c_1g(g) \leq f(n)\} \)

\(o(g(n))= \{f(n) : \forall c_1 > 0,\exists n_1 > 0 \bigwedge n \geq n_1\rightarrow 0 \leq f(n) < c_1g(n)\} \)

\(\omega(g(n))= \{f(n) : \forall c_1 > 0,\exists n_1 > 0 \bigwedge n \geq n_1\rightarrow 0 \leq c_1g(g) < f(n)\} \)
